# 代码审查质量规则

## 通用规则
- 所有公共函数必须有 JSDoc/docstring 注释
- 禁止使用 any/unknown 类型（TypeScript）
- 所有异步操作必须有错误处理
- 禁止在循环中执行 await（除非有明确并行需求）
- 环境变量必须通过配置层访问，禁止硬编码

## 安全规则
- SQL 查询必须使用参数化查询
- 用户输入必须经过校验和转义
- 密码必须使用 bcrypt/argon2 哈希
- JWT 必须设置过期时间
- API 端点必须有认证和授权检查

## 性能规则
- 数据库查询必须使用索引
- 禁止在热路径中执行同步 I/O
- 大列表操作必须使用分页
- 缓存必须有 TTL 和淘汰策略
