# 安全审查专项规则

## 必须检查的项目
1. 用户输入是否经过校验和转义
2. SQL/NoSQL 查询是否参数化
3. 文件路径操作是否有路径遍历防护
4. 加密操作是否使用标准库和推荐算法
5. 认证 token 是否有适当的过期和刷新机制
6. 敏感数据是否在日志中脱敏
7. CORS 配置是否过于宽松
8. 依赖项是否有已知安全漏洞

## 误报过滤规则
以下情况应标记为误报并跳过：
- 非 C/C++ 文件的内存安全问题
- 速率限制和 DOS 防护建议（除非有明确漏洞）
- 开放重定向（低风险场景）
- 代码生成文件（@generated 标记）
- 测试文件中的 low 级别安全建议
- TODO/FIXME 注释
- 日志级别建议
